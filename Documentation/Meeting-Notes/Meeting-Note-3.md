- เปลี่ยนชื่อจาก "C-REWARDS" เป็น "The Trophy"
- 1.3 Implementation Approach
	- ช่วยอธิบายเพิ่มหน่อย คือลูกค้าเขาตั้ง Phase มาใหม่เหมือนเราของเขาจะมี 
		1. MVP
		2. Automation / Scaling 
		3. Ecosystem Expansion 
	- เราไม่ต้องล้อตามเขา แต่ต้องอธิบายว่าเราเพิ่ม POC ไปก่อนจะถึง MVP เพราะว่าอะไร และ "Automation / Scaling" กับ "Ecosystem Expansion" เรารวมอยู่ใน "Full-Scale Ecosystem" ของเราแล้ว 
	- เป้าหมายคือเพื่อให้เราไม่ต้องแก้เอกสารทั้งหมด และอยากให้ลูกค้ารู้ว่าเราได้อ่านเนื้อหาของเขาเรียบร้อยแล้ว 
- ลูกค้าใช้คำว่า "Trophy Wallet" ซึ่งของเราใช้แนวๆ "Awards Point" (มั้ง) ซึ่งมันยังไม่ได้มีจุดที่บอกว่ามันทำงานเสมือนเป็น Wallet ตามที่ลูกค้าต้องการในนิยามของ "Trophy Wallet" นั่นแหละ หาจุดที่บอกให้หน่อยแล้วกัน
- ลูกค้าบอกให้เราใช้ AI ในการตรวจหลักฐานด้วยนะ เช่น เรื่องของการถ่ายภาพตามสถานที่ต่างๆ แล้วให้ AI มาดูวัตถุ ซึ่งดูแล้วมันสิ้นเปลืองมาก กูเห็นว่าการใช้ Staff ของ Partner ที่หน้างาน จะดูคุ้มและได้ปฏิสัมพันธ์กับ End User มากกว่า, ช่วยหาที่ชี้แจงเรื่องนี้ไว้ให้หน่อยนะ
- ลูกค้าอยากให้ End user เข้าใช้งานผ่าน "เบอร์มือถือ หรือ Social Login" ซึ่งกูมองว่ามันควรจะเข้าด้วยเบอร์มือถืออย่างเดียวก็พอ (ช่วยกูอ้างด้วยนะ) แล้วก็หาจุดเพิ่มระบบเปลี่ยนเบอร์มือถือให้นิดหน่อยแล้วกันลูกค้าจะได้ไม่กังวล 
- เพิ่มฟีเจอร์เด็ด Point Transfer เพื่อเป็นการเชื่อมต่อกับระบบสะสมแต้มของ Partner ต่างๆที่เราจะร่วมมือกับเขาในอนาคต ไม่ว่าจะเป็น Loyalty Program ของ Partner หรือการแลกแต้มจากบัตรเครดิต ได้ทั้งขาไปและขากลับ 
- ลูกค้าต้องการให้มีฟีเจอร์ 
- ลูกค้าอยากให้มีระบบ 
	- ส่งแต้มให้เพื่อน ส่วนตัวกูมองว่ามันทำได้ แต่มันจะทำให้ Point Economy มีปัญหา ฝากบอกเขาให้หน่อยแล้วกัน
